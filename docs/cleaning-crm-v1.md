# 面向本地小型保洁公司的极简 CRM（客户跟进 & 续费提醒）V1 功能说明

## 1. 产品定位

- **产品形态**：云端 Web 应用（后端 + Web 前端，手机浏览器可用为主）
- **目标用户**：本地小型保洁公司老板/店长（3–30 人团队），数字化能力一般
- **核心价值**：
  - 不再漏掉客户咨询和老客户续费机会
  - 每天/每周清楚知道该联系哪些客户
  - 对客户有简单的“台账”，而不是散落在微信、纸质笔记、Excel 里

> V1 只解决：**客户记录 + 跟进提醒 + 续费提醒 + 简单统计**。
> 不做派单、工资结算、考勤等复杂家政/ERP 功能。

---

## 2. 核心业务流程（V1 范围）

### 2.1 客户生命周期（简化版）

1. 客户咨询（电话/微信/平台）
2. 录入客户信息（基础信息 + 服务需求）
3. 安排并完成保洁服务
4. 服务后回访（满意度 & 备注）
5. 到期前续费提醒（包月/周期性服务）
6. 流失客户唤回（很久未服务）

V1 重点支持步骤：**2 / 3 / 4 / 5 / 6**。

---

## 3. 角色与权限（V1）

- **公司（租户）**
  - V1 默认一个系统支持多公司（多租户），用 `company_id` 区分数据。

- **用户角色**
  - **老板/管理员**（V1 只实现这一种角色）：
    - 可以管理本公司的所有客户与服务记录。
    - 可以查看简单统计数据。
  - 其他角色（客服/业务员/保洁员）在后续版本再考虑，不在 V1 范围内。

---

## 4. 页面与功能列表

### 4.1 登录 / 退出

- **登录**（V1 可选实现，优先度中）：
  - 使用手机号 + 密码登录，或简单账号密码。
  - 登录后只能访问所属公司的数据。
- **退出登录**：
  - 清除会话/Token，返回登录页。

> 若 V1 内部仅供开发和少量种子用户使用，可临时使用固定测试账号，正式登录流程可延后实现。

---

### 4.2 客户列表页

**目标**：一眼看到主要客户信息，支持简单筛选。

- **展示字段（表格形式）**：
  - 客户姓名
  - 手机号
  - 小区/地址（例如“XX 小区 XX 栋”）
  - 服务类型（枚举，如：一次性保洁 / 包月保洁 / 深度保洁）
  - 最近一次服务日期
  - 下次建议服务日期（用于生成提醒）
  - 客户状态：
    - 新客户 / 正在跟进 / 已成单 / 长期客户 / 已流失

- **基础操作**：
  - 新增客户
  - 编辑客户
  - 查看客户详情

- **筛选/排序（V1 简化）**：
  - 按客户状态筛选
  - 按小区模糊搜索
  - 按最近服务日期排序（最近在前）

---

### 4.3 客户详情页

**目标**：集中查看单个客户的完整信息以及历史服务记录。

- **客户基本信息**：
  - 姓名
  - 手机号
  - 地址/小区
  - 备注（自由文本，比如家庭情况、偏好等）

- **服务偏好与类型**：
  - 服务类型（一次性 / 包月 / 深度保洁 等）
  - 期望服务频率（例如每周一次、每两周一次）

- **历史服务记录列表**：
  - 每条记录包含：
    - 服务日期
    - 服务类型
    - 简要备注（比如“厨房油烟重、加时半小时”）
    - 回访结果（满意 / 一般 / 不满意，可选）

- **下次服务/续费信息**：
  - 下次建议服务日期
  - 是否开启续费提醒（勾选项）

---

### 4.4 服务记录管理

**目标**：为客户增加服务记录，让系统知道“什么时候服务过”。

- **新增服务记录**：
  - 选择客户（从客户列表中选择或在详情内直接添加）
  - 填写：
    - 服务日期（默认今天，可修改）
    - 服务类型
    - 备注（可选）
    - 是否需要回访 & 回访时间（例如服务后 3 天）

- **服务结束后的结果更新**（可与回访合并）：
  - 回访时更新：
    - 回访日期
    - 满意度（满意 / 一般 / 不满意）
    - 客户反馈简要备注

> V1 可不做复杂的工单/派单流程，只把“服务记录”当作简单的历史记录和回访触发器。

---

### 4.5 今日/本周待办提醒列表

**目标**：让老板每天打开系统，就知道“今天要联系谁”。

- **待办类型**：
  - 到期前续费提醒：
    - 针对包月/周期性客户，根据下次建议服务日期提前 N 天提醒（例如 7 天）。
  - 回访提醒：
    - 基于服务记录里设置的“回访时间”。

- **页面展示**：
  - Tab 或筛选：
    - 今日待办 / 本周待办 / 所有未处理任务。
  - 每条待办显示：
    - 客户姓名 + 手机号
    - 小区
    - 待办类型（续费 / 回访）
    - 相关服务日期/下次建议服务日期

- **处理操作**：
  - 标记为已完成
  - 添加备注（例如“已续费一个月”“电话打不通”）

> V1 不做自动发短信/微信，只做“提醒列表 + 手动记录结果”。

---

### 4.6 简单统计页

**目标**：让老板对当前业务状况有一个大致感觉，不追求复杂报表。

- **时间维度**：
  - 默认展示“本月”，可切换“上月”。

- **指标（示例）**：
  - 本月服务的客户数量
  - 本月新增客户数量
  - 本月有过服务但未续费且已过期的客户数量（潜在流失）
  - 本月回访过的客户数量

> 统计维度可基于服务记录和客户状态简单汇总，V1 控制在 3–5 个关键数字。

---

## 5. 非目标（V1 不做的功能）

为避免范围失控，V1 明确**不做**以下功能，仅作为后续版本候选：

- 派单/排班系统（按员工/阿姨排班）
- 员工/阿姨管理（档案、结算、考勤等）
- 财务结算、工资计算、利润分析
- 复杂权限体系（如区分客服、经理、保洁员权限）
- 自动发送短信/微信通知（可以在 V2 之后再考虑接入短信或企微/公众号）
- 多语言、多城市复杂运营

---

## 6. 技术侧约束（简要）

- **多租户设计**：
  - 所有与数据相关的主表（用户、客户、服务记录、待办任务）需要带 `company_id` 字段，以支持多个保洁公司共用一套系统。

- **移动端友好**：
  - 虽是 Web 应用，但页面布局要适配手机浏览器使用场景，列表和表单尽量简洁。

- **数据可导出**（可选）：
  - 至少在客户列表支持导出为 CSV/Excel，方便老板有“可带走的数据”安心感。

---

## 7. V1 实现优先级建议

- **优先级高（必须实现）**：
  - 客户列表 + 新增/编辑/详情
  - 服务记录新增 & 查看
  - 今日/本周待办提醒列表（续费 & 回访）
  - 基础多租户结构（company_id）

- **优先级中（尽量在 V1 完成）**：
  - 简单统计页
  - 登录/退出（支持区分不同公司用户）

- **优先级低（可推迟）**：
  - 数据导出
  - 更复杂的筛选和报表

以上作为 V1 的基准功能范围，后续可根据真实用户反馈进行微调和迭代。
